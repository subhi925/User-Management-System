# Test Plan

## 1. Objective
To verify that the User Management System works correctly according to the defined requirements, including authentication, authorization, and user management.

## 2. Scope
The testing will include:
- Registration and login
- User and Admin dashboards
- Role-based access control
- User profile management
- Admin user management

## 3. Test Types
- Functional testing
- Negative testing
- Authorization and access control testing
- UI validation

## 4. Test Environment
- Web application
- Google Chrome browser
- Firebase Authentication
- Firestore database

## 5. Entry Criteria
- Application is deployed and accessible
- Test data is available (Admin and User accounts)

## 6. Exit Criteria
- All planned test cases are executed
- Critical bugs are fixed or reported

## 7. Out of Scope
- Performance testing
- Security penetration testing
- Mobile testing
